## 🔧 手动操作步骤（需要管理员权限）

### 步骤1: 停止QClaw相关进程
**目的**: 确保目录不被占用  
**操作**: 
1. 退出QClaw应用程序
2. 打开任务管理器，结束以下进程（如果存在）：
   - `openclaw.exe`
   - `node.exe`（QClaw相关）
   - `gbrain.exe`

### 步骤2: 删除C盘原始目录
**风险**: **删除后无法恢复！**  
**操作**: （在管理员CMD中执行）

```cmd
# 删除.openclaw原始目录（已备份为.openclaw.backup）
rmdir /S /Q C:\Users\Administrator\.openclaw

# 删除.agentmemory原始目录（已备份为.agentmemory.backup）
rmdir /S /Q C:\Users\Administrator\.agentmemory

# 删除.agents原始目录（未被重命名，需直接删除）
rmdir /S /Q C:\Users\Administrator\.agents
```

**预期结果**: 3个原始目录被删除，仅保留.backup备份

### 步骤3: 创建符号链接
**操作**: （在管理员CMD中执行）

```cmd
# 创建.openclaw符号链接
mklink /J "C:\Users\Administrator\.openclaw" "D:\QClawX\.openclaw"

# 创建.agentmemory符号链接
mklink /J "C:\Users\Administrator\.agentmemory" "D:\QClawX\.agentmemory"

# 创建.agents符号链接
mklink /J "C:\Users\Administrator\.agents" "D:\QClawX\.agents"
```

**预期结果**: 3个符号链接创建成功，指向D:\QClawX下对应目录

### 步骤4: 验证符号链接
**操作**: 

```cmd
# 检查链接类型
fsutil reparsepoint query C:\Users\Administrator\.openclaw
fsutil reparsepoint query C:\Users\Administrator\.agentmemory
fsutil reparsepoint query C:\Users\Administrator\.agents

# 测试链接是否工作
dir C:\Users\Administrator\.openclaw
dir C:\Users\Administrator\.agentmemory
dir C:\Users\Administrator\.agents
```

**预期结果**: 
- `fsutil`输出显示"Printf reparse tag: Junction"（说明是链接）
- `dir`命令显示D:\QClawX下对应目录的内容

### 步骤5: 删除D盘重复数据（可选，节省空间）
**前提**: 确保符号链接工作正常，QClaw能正常访问D:\QClawX下的数据  
**风险**: 删除后无法恢复，必须确认数据已完整合并  

