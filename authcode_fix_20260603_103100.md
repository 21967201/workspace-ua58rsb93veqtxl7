`task_push.py` 从 QClaw 全局配置读取 `authCode` 和 `pushServiceUrl`，但全局配置 `C:\Users\Administrator\.qclaw\qclaw.json` 中 `skills.entries.today-task` 为空对象，导致读取失败。

## 修复内容

### 1. 修复全局配置（一劳永逸）
- 文件：`C:\Users\Administrator\.qclaw\qclaw.json`
- 操作：添加 `skills.entries.today-task.config` 节点
- 结果：全局配置现在包含正确的 `authCode` 和 `pushServiceUrl`

### 2. 修复本地 config.json 编码
- 文件：`D:\QClawX\data\workspace\skills\today-task\config.json`
- 问题：UTF-8 with BOM 导致 Python 读取失败
- 修复：重写为 UTF-8 without BOM

### 3. 修改 `task_push.py` 添加多重 Fallback（永久方案）
修改函数 `load_qclaw_config()`，添加 4 层 fallback：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1 | 本地 `config.json` | 最可靠，优先使用 |
| 2 | 全局 QClaw 配置 | 次优先 |
| 3 | 环境变量 | `TODAY_TASK_AUTH_CODE` / `TODAY_TASK_PUSH_URL` |
| 4 | 硬编码默认值 | 最后防线，authCode=`dtG4JOCLM3ev` |

**这样无论哪种配置方式失败，都能正常工作。**

## 防止再次犯错的措施

### 措施 1：修改 `task_push.py` 添加配置自检
在 `push()` 函数开头添加：
```python
# 配置自检
auth_code, push_url = load_qclaw_config()
if not auth_code or not push_url:
    return {"success": False, "message": f"配置未找到！authCode={'已配置' if auth_code else '未配置'}, pushServiceUrl={'已配置' if push_url else '未配置'}"}
```

### 措施 2：添加 cron 任务执行前配置检查
在 cron 任务开头添加：
```powershell
# 配置检查（必须！）
$configCheck = python -X utf8 -c "import sys; sys.path.insert(0, r'D:\QClawX\data\workspace\skills\today-task\scripts'); from task_push import load_qclaw_config; auth, url = load_qclaw_config(); print('AUTH_OK' if auth else 'AUTH_MISSING', flush=True); print('URL_OK' if url else 'URL_MISSING', flush=True)"
if ($configCheck -match 'MISSING') {
    Write-Error "配置检查失败！停止执行。"; exit 1
}
```
