# 任务总结 - today-task技能配置问题修复

**时间**: 2026-05-29 11:15  
**任务**: 修复today-task技能无法读取authCode配置的问题

## 问题现象

执行用户提供的3个配置命令后，today-task技能仍然无法读取`authCode`：
```bash
npx clawhub@latest install today-task --force --registry=https://mirror-cn.clawhub.com
openclaw config set skills.entries.today-task.config.authCode dtG4JOCLM3ev
openclaw gateway restart  # 这个命令失败，但不是真正的问题
```

测试读取配置时返回：`authCode: None`

## 真正的问题根源

**用户提供的3个命令完全正确！** 问题出在`today-task`技能的代码中。

`D:\QClawX\data\workspace\skills\today-task\scripts\config.py`的`_get_from_openclaw_global()`方法有**逻辑BUG**：

1. 代码先正确读取了`C:\Users\Administrator\.qclaw\openclaw.json`（QClaw配置）
2. **但第108-117行错误地覆盖了读取结果**：
   ```python
   if os.path.exists(openclaw_config_path):  # 这行检查.openclaw路径
       with open(openclaw_config_path, 'r', encoding='utf-8') as f:
           openclaw_config = json.load(f)  # 这行覆盖了正确的配置！
   ```
3. 如果`.openclaw\openclaw.json`不存在或没有`authCode`，就返回`NOT_SET_IN_OPENCLAW`

## 修复方法

删除`config.py`第103-117行的错误覆盖逻辑，让代码只读取一次配置：

```python
# 读取配置文件
with open(config_path, 'r', encoding='utf-8') as f:
    openclaw_config = json.load(f)

# 获取技能配置
skill_config = openclaw_config.get('skills', {}).get('entries', {}).get('today-task', {})

# 获取config中的值
config_value = skill_config.get('config', {}).get(key)

if config_value:
    logger.debug(f"从OpenClaw全局配置读取 {key}: {config_value[:4]}***" if key == 'authCode' else f"从OpenClaw全局配置读取 {key}: {config_value}")
    return config_value
else:
    logger.debug(f"OpenClaw全局配置中未找到 {key}")
    return 'NOT_SET_IN_OPENCLAW'
```
