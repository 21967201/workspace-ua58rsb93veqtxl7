# GBrain 云端记忆增量同步任务设置

## 任务目标
为用户设置自动化任务，每周自动更新云端记忆，实现增量更新（只更新有变化的章节）。

## 关键推理

### 1. 需求分析
- **核心需求**: 每周自动更新云端记忆
- **关键要求**: 增量更新（只更新有变化的章节）
- **时间要求**: 每周执行一次

### 2. 技术选型
- **云端记忆系统**: GBrain 知识库（本地知识库 + Git 版本控制）
- **增量更新机制**: 
  - GBrain 的 `import --no-embed` + `embed --stale` 实现文件级增量
  - Git 的 `add -A` + `commit` + `push` 实现版本控制级增量
- **自动化调度**: OpenClaw Cron 任务系统

### 3. 实现方案
创建每周一 15:00 执行的 Cron 任务，执行以下增量同步流程：
1. **检查 Git 仓库状态** - 如果未初始化则初始化
2. **执行 GBrain 增量导入** - 只导入变化的文件
3. **增量生成向量嵌入** - 只为变化的文件生成嵌入
4. **检测文件变化** - 使用 `git status --porcelain`
5. **增量提交** - 只提交变化文件
6. **推送到远程仓库** - 如果已配置远程仓库
7. **生成同步报告** - 记录同步结果
8. **推送结果到负一屏** - 使用 today-task 技能

## 技术细节

### 增量更新机制
1. **GBrain 级别增量**:
   - `bun run src/cli.ts import <path> --no-embed` - 只导入变化的 Markdown 文件
   - `bun run src/cli.ts embed --stale` - 只为变化的文件生成向量嵌入

2. **Git 级别增量**:
   - `git status --porcelain` - 检测变化文件
   - `git add -A` + `git commit -m <msg>` - 只提交变化文件
   - `git push origin master` - 只推送变化提交

3. **报告级别增量**:
   - 只记录变化文件列表
   - 只生成包含变化内容的报告

### 错误处理
- **GBrain 导入失败**: 记录错误，继续后续步骤
- **向量嵌入失败**: 记录错误，继续后续步骤
- **Git 提交失败**: 可能无变化，跳过推送
- **远程推送失败**: 记录错误，继续后续步骤
- **无文件变化**: 任务提前结束，不生成报告
